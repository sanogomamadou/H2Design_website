2. **Asymmetric Monograph Grid & Spatial Rhythm**: The project section breaks grid monotony with desktop stagger offsets (`row-left`, `row-right`, `row-featured`), while the footer balances a colossal 14rem watermark (`H2 DESIGN`) with a pristine 4-column layout, creating an editorial feel.
3. **Interactive Polish & Accessibility**: Flawless implementation of 'Flat at Rest, Lifted on Action' micro-interactions, robust keyboard navigation (`ArrowLeft`/`ArrowRight`), and full support for `prefers-reduced-motion` ensure an elite, accessible user experience.

#### Priority Issues
- **[P1] Simulated Espace Client Access Button (`dt-btn-primary`)**: Currently triggers a JavaScript `alert()` (`Simulation Espace Client Sécurisé : En cours de déploiement`). While transparent, an alert dialog can feel abrupt to high-intent diaspora users seeking immediate portal login.
  - **Why it matters**: Breaks brand immersion for high-value leads looking to inspect their remote construction ledger.
  - **Fix**: Replace the `alert()` with a beautiful, custom in-page modal dialog or a dedicated slide-over drawer that captures their email/WhatsApp to notify them the moment the secure portal V2 is deployed.
  - **Suggested command**: `impeccable polish`

- **[P2] Popup Blocker Vulnerability on WhatsApp Form Redirection**: The contact form uses `window.open(waUrl, '_blank')` inside a `setTimeout`. Aggressive browser popup blockers often block async `window.open` calls not directly tied to a synchronous click event.
  - **Why it matters**: Can stall the user's transition to WhatsApp if their browser suppresses the new tab.
  - **Fix**: Enhance the success panel (`form-success`) by adding a prominent, one-click 'Copier le message' (Copy to clipboard) button alongside the manual 'Démarrer la discussion' link, ensuring 100% fail-safe reliability.
  - **Suggested command**: `impeccable polish`

- **[P2] Static Fallback for Live Bamako Time Indicator**: The `getBamakoTimeStr` function uses `Intl.DateTimeFormat` with `timeZone: 'Africa/Bamako'`. In environments where specific African IANA timezones are outdated or unsupported, the `catch` block falls back to raw UTC hours/minutes.
  - **Why it matters**: During daylight savings changes in other regions, this could cause slight confusion if not explicitly labeled as UTC/GMT+0.
  - **Fix**: Add a robust, lightweight fallback validation in `src/main.js` that explicitly verifies the GMT offset calculation for Bamako (always UTC+0) and formats the display string with a reassuring tooltip explaining Mali's fixed GMT+0 timezone.
  - **Suggested command**: `impeccable polish`

#### Persona Red Flags
**Alex (Power User / Diaspora Investor)**: No quick-access keyboard shortcuts detected (e.g., pressing `C` to jump directly to the contact form or `P` for projects). Adding a subtle keyboard shortcut listener would delight power users managing large investment portfolios.

**Jordan (First-Timer / Remote Family Member)**: The simulated `alert()` on the Espace Client button might cause momentary confusion for a first-timer who expects a traditional login screen. Furthermore, if their browser blocks the automated WhatsApp popup after form submission, they might overlook the manual link in the success box if they scroll away too quickly.

#### Minor Observations
- The testimonial marquee track pauses flawlessly on hover/focus, but adding `aria-hidden="true"` to the duplicated marquee elements (`tabindex="-1"`) ensures screen readers don't read duplicate testimonial copy.
- The WhatsApp floating button (`whatsapp-float`) has a fixed z-index of `990`, which layers correctly but should be tested against mobile viewport keyboards to ensure it doesn't occlude submit buttons on smaller screens.
