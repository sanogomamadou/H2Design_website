## Detailed Findings by Severity

### [P1] Mobile Menu Focus Trap & Scroll Lock Gap
* **Location:** Mobile Navigation Menu (`src/main.js:25-50`, `src/style.css:2560-2617`)
* **Category:** Accessibility
* **Impact:** Disorienting keyboard/screen-reader navigation. Users can Tab out of the visual overlay sidebar and interact with page content hidden behind the menu.
* **WCAG/Standard:** WCAG 2.1 Success Criterion 2.4.3 (Focus Order) & 1.3.2 (Meaningful Sequence)
* **Recommendation:** Implement a mobile menu focus trap in `src/main.js` similar to the one built for the project drawer. Toggle `document.body.style.overflow = 'hidden'` when the menu is opened/closed.
* **Suggested Command:** `/impeccable harden`

### [P1] Secondary Project Cards Grid Mobile Collapse Failure
* **Location:** Realisations Section (`src/style.css:2422-2432`)
* **Category:** Responsive Design
* **Impact:** Squeezes the secondary columns (`row-left` and `row-right`) into a tiny, narrow layout on screen widths under 768px, wrapping titles and info into unreadable blocks.
* **WCAG/Standard:** WCAG 2.1 Success Criterion 1.4.10 (Reflow)
* **Recommendation:** Update mobile media queries in `src/style.css` to collapse all project card grid columns (`.projects-grid`) to a single column (`1fr`) on viewports under 1024px.
* **Suggested Command:** `/impeccable adapt`

### [P2] Featured Card Side-by-Side Layout Overlap on Mobile
* **Location:** Realisations Section (`src/style.css:1106-1111` & `src/style.css:1138-1165`)
* **Category:** Responsive Design
* **Impact:** Description text and specifications table remain side-by-side on mobile viewports, causing squished texts, overlapping layout, and clipping.
* **WCAG/Standard:** WCAG 2.1 Success Criterion 1.4.10 (Reflow)
* **Recommendation:** Reset the `.p-meta-grid` grid-template-columns property to `1fr` in the media query, forcing the specs table to stack neatly below the title and description.
* **Suggested Command:** `/impeccable adapt`

### [P2] Testimonials Marquee Clipping and Control Gap
* **Location:** Testimonials Section (`src/style.css:1522-1544`, `src/style.css:1637-1666`)
* **Category:** Responsive Design / Accessibility
* **Impact:** Testimonial cards are clipped on mobile screen edges without horizontal scrollbars, touch-swipe support, or pagination buttons, making parts of the feedback unreadable.
* **WCAG/Standard:** WCAG 2.1 Success Criterion 2.1.1 (Keyboard - must pause on focus) & 1.4.4 (Resize text)
* **Recommendation:** Provide standard manual scroll overflow or simple horizontal slider controls on mobile viewports instead of relying purely on CSS infinite translation.
* **Suggested Command:** `/impeccable adapt`

### [P2] Layout Thrashing in Scrollspy Listener
* **Location:** Scrollspy Navigation (`src/main.js:675-682`)
* **Category:** Performance
* **Impact:** Calling `footer.getBoundingClientRect().top` on every scroll event forces synchronous layout recalculation, dropping frames and causing scroll stutter on low-end hardware.
* **WCAG/Standard:** Browser Rendering Performance (Avoid layout-thrashing API reads on scroll)
* **Recommendation:** Cache the footer's `offsetTop` value during initialization and resize (just like the section offsets), and query the cached value instead of calling the DOM API on every scroll event.
* **Suggested Command:** `/impeccable optimize`

### [P3] Cumulative Layout Shift (CLS) from Missing Image Dimensions
* **Location:** Image Tags (`index.html` lines 52, 54, 55, 88, 98, 117, etc.)
* **Category:** Performance
* **Impact:** Browsers cannot reserve space for images before they finish downloading, causing sudden content shifts as images render.
* **WCAG/Standard:** Web Vitals (CLS < 0.1 target)
* **Recommendation:** Add explicit `width` and `height` attributes to the HTML `<img>` elements alongside the CSS layouts.
* **Suggested Command:** `/impeccable polish`

## Patterns & Systemic Issues
* **Hard-coded visual borders:** Borders (`#eee`, `#ccc`, `#e0e0e0`) and background colors (`#fafafa`) are defined locally across 20+ places in CSS. These should be moved to CSS Custom Properties (e.g. `--c-border-light`, `--c-bg-neutral`).
* **Inconsistent mobile grid rules:** Grid layouts collapse to single-columns in some sections (About, Services) but fail to collapse or stack properly in others (Projects, Testimonials), indicating a lack of unified mobile layouts.
